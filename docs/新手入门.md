# 私域专家团新手入门

## 一、先记住一个入口

输入 `/siyu`，或直接说你现在的真实处境。你不需要先判断该用哪个 skill，也不用按固定流程从头跑到尾。

可以这样开始：

```text
朋友圈每天都在发，但来咨询的人越来越少，我现在该先改内容还是改人群？
```

主入口会读取已有上下文，每次只选择当前一个能力。完成后想继续，再输入 `/siyu`，它会根据实际结论重新导航。

## 二、你会经历的四步

1. 说出现状、目标或材料，信息不完整也可以。
2. 系统先把请求整理成任务类型、渠道、目标和风险，再由 `/siyu` 选择当前最值得处理的一步。
3. 执行能力直接产出文案、话术、诊断或 playbook。
4. 有结论就保存客户档案；要继续就重新导航。

日常动作优先走执行能力。只有具体问题活过问诊漏斗、且涉及全盘结构，才升舱四官评审。

## 三、完整能力目录

| skill | 适用时机 | 输入示例 | 常见衔接 |
|---|---|---|---|
| `/siyu` | 第一次使用、不知道从哪开始、任务做完想找下一步 | `/siyu`；「下一步怎么走」 | 路由到下面任一能力 |
| `/siyu-pyq` | 写朋友圈、节日内容、整周内容池 | 「给门店写 7 条周末朋友圈」 | 群发、轻问诊、存档 |
| `/siyu-qunfa` | 群发通知、社群栏目、活动推送 | 「写一条周五晚的限时券群发」 | 话术承接、轻问诊、存档 |
| `/siyu-huashu` | 新客欢迎、加好友破冰、高频答疑 | 「写新客扫码后的第一句话」 | 朋友圈、群发、全盘诊断 |
| `siyu-wenzhen` | 转化差、没人加微、留存掉等具体问题 | 「群不活跃怎么办」 | 对应执行能力或 `siyu-onboard` |
| `siyu-onboard` | 全盘诊断、活动战略评审、整盘搭建 | 「帮我看整个私域盘子」 | 执行能力或 `/siyu-save` |
| `/siyu-save` | 已有结论，想下次接着用 | 「把这个结论存档」 | `/siyu-restore`；三份后报告 |
| `/siyu-restore` | 接着上次、查看之前否决过什么 | 「上次聊到哪了」 | 档案中的下一步或 `/siyu` |
| `/siyu-report` | 多次咨询后要交付给老板或客户 | 「把这个客户的记录整理成报告」 | 新一轮问诊或执行 |
| `/siyu-update` | 更新本工具箱 | 「更新私域专家团」 | 更新后新建对话 |
| `trust-asset-playbook` | 专家团内部评审信任与口碑资产 | 「检查这套活动如何积累信任」 | 由团长收口 |
| `crisis-response-sop` | 专家团内部评审负面与危机响应 | 「给这次投诉设计响应 SOP」 | 由团长收口 |
| `content-as-product` | 专家团内部评审内容产品性 | 「检查这批内容是否值得看和复用」 | 由团长收口 |
| `reactivation-playbook` | 专家团内部评审沉默用户唤醒 | 「设计沉默 30 天用户的唤醒方案」 | 群发或话术执行 |
| `ops-as-ad-funnel` | 专家团内部评审触达与转化漏斗 | 「拆这次活动的完整漏斗」 | 由团长收口 |
| `conversion-caliber` | 专家团内部校准指标口径 | 「把转化率的分子分母和时间窗写清」 | 质量门与数据埋点 |
| `wechat-compliance-redlines` | 专家团内部审查封号、广告与授权风险 | 「检查这套裂变动作是否合规」 | 红线先整改，再收口 |

普通用户不必记住后七个内部评审 skill。需要全盘诊断时调用 `siyu-onboard`，团长会组织它们。

## 四、三个常见用法

### 日常内容

```text
/siyu
这周要推一个真实的会员活动，先帮我排朋友圈。
```

完成后如需社群触达，再输入 `/siyu`。主入口会依据文案结论决定是否接 `/siyu-qunfa`，不会提前替你跑长链。

### 带着具体问题来

```text
我的群不活跃怎么办？
```

`siyu-wenzhen` 会先让你定义“活跃”是发言、打开、核销还是复购。问题没说清前，不会直接扔一份促活清单。

### 跨对话客户咨询

```text
把这次结论存档。
```

下次说「接着上次」，系统会恢复主诉、已确认结论、已否决方向和推荐下一步。累计多份后可生成合规 markdown 报告。

## 五、本地记录与隐私

- 客户档案写在 `~/.siyu/clients/`，报告写在 `~/.siyu/reports/`。
- 这些文件是本地纯文本，不加密；敏感信息应最小化保存。
- `.siyu-team/` 保存一轮任务的原子状态和 JSONL 追踪，不等于长期客户档案。
- 追踪写盘前会脱敏常见凭据、手机号和身份证号；业务正文仍应坚持最小化收集。
- 更新工具箱不会触碰 `~/.siyu/`。

## 六、合规与真实语料

- 文案、群发和话术在生成端先做合规扫描。
- 质量门命中 `COMPLIANCE_RED` 时直接停止，不交付。
- 没接真实原子库时，执行产物会标注为通用示例。
- 真实 SOP 和客户数据不会被自动编造，也不会进入公开仓库。

## 七、安装与更新

公开仓库可运行：

```bash
npx -y skills add maojiebc/majia-siyu-team -g --all
```

私有期从本地安装：

```bash
npx -y skills add /path/to/majia-siyu-team -g --all
```

安装后新建一次对话，输入 `/siyu 新手入门`。以后要更新，直接说「更新私域专家团」。
