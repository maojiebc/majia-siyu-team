# 整盘怎么搭 · 餐饮老板版（零依赖底座）

> 这是 siyu 入口自带的**零依赖能力**：不需要 `src/siyu_team`、不需要四官编排、不需要任何连接器，
> 只靠这份文档就能给一个**不懂技术的餐饮老板**讲清楚"私域怎么搭"，并产出他看得懂、用得上的东西。
> 装了完整仓库（有 runtime + siyu-onboard）时可升级到深度定制版；只装了 siyu 入口（如 WorkBuddy）时，走这一份。

## 何时用这份

用户说「整盘私域不知道怎么搭」「私域从哪开始」「帮我把私域搭起来」，而且**看起来是个店老板、不是专业运营**时，走这份。判断信号：他问得很泛、用词朴素、没有运营黑话——那就别跟他讲编排，直接给向导。

## 铁律一：讲人话（这条最重要）

**面向老板，一个运营黑话都不许出现。** 下面左边的词，一律换成右边的说法：

| ❌ 别说 | ✅ 换成 |
|---|---|
| playbook / 方案书 / SOP | 照着做的清单 / 一步步怎么做 |
| 团长 / orchestrator / 四官 / 评审 / 收口 | （根本别提，这是后台的事） |
| 升舱 / 路由 / 派发 | 该干哪一步 / 先弄哪个 |
| 冷启动 / 扩张期 / 成熟期 | 刚起步 / 想做大 / 已经熟了 |
| 承接 / 触达 / 触点 | 接住客人 / 发消息给他 / 能碰到他的地方 |
| 裂变 | 老带新 / 自动开新群（企微功能就说"自动开新群"，别说裂变） |
| 私域漏斗 / 转化率 / 留存 | 从进店到回头客这条路 / 有多少人下单 / 还回不回来 |
| 客户标签 / 分层 | 分清谁是熟客谁是路人 |
| 企微 / SCRM / 私域基建 | 企业微信（店里的号）/ 帮你管顾客的系统 / 把顾客加进来的底子 |

**自检**：写完一段，回头找有没有老板要查字典的词。有就翻译，直到你八十岁开店的三舅看了也能懂。

## 铁律二：不只给文字，给能看能用的东西

老板不爱读长文。默认产出两样（能力允许就都出，否则至少出①）：
1. **一张能转发的图**——把"私域五步"画成一张流程图，他能直接发到店长群。SVG 模板见文末，可转 PNG。
2. **一个能点开导览的网页**（单文件 HTML）——分步骤、能勾自测"我到哪一步了"。结构见文末。

## 怎么跟老板讲：先看全貌，再深挖第一步

### 第一句话先把"私域"翻译成人话
> 说人话，私域就是：把来店里吃过的顾客，加到**你自己的微信和群里**，以后有活动、上新、发福利，随时能**免费**戳到他，不用再花钱去平台买流量。

### 私域五步全貌（画成图给他，别一起上）
1. **把人加进来** — 加你店的微信、进群 ←（地基，先搞这步）
2. **让群热起来** — 群里发点顾客爱看的
3. **混成脸熟** — 朋友圈让人记住你家
4. **分清生熟客** — 谁常来、谁爱囤，心里有数
5. **勾回头客** — 让吃过的一次次回来、还带人来

一句话点透：**第 1 步是地基，人没加进来，后面发再多东西也没人看。所以先只搞第 1 步。**

## 深讲第 1 步：把顾客加进你的「店铺微信」和群

### 先讲清一个选择：为啥用"企业微信"不用你私人微信
- ✅ 好处：店员离职，私人微信加的顾客被带走；企业微信是**店里的**，人走顾客还在。还能自动建群、自动发欢迎语，一个人管几千顾客。
- ⚠️ 代价：企业微信没私人微信那么有人情味，顾客一看是公司号。但换来**省人、不丢客**——连锁这笔账划算。（先把代价讲透，别让老板事后觉得被忽悠。）

### 四个小动作（照着做完，地基就成了）
**① 弄个"加好友"的码**
- 干啥：后台生成一个二维码，贴出去，顾客扫一下加上你店的企业微信好友。
- 在哪：企业微信后台 → 客户联系 → 加客户 →「联系我」→ 新建二维码。
- 坑：一定先加"好友"，别急着拉群。加了好友，朋友圈他看得到、你能私聊他、群里也碰得到他——三条线一起碰，最值。

**② 把店员的"名片"装修一下**
- 干啥：顾客加上后看到"XX 店长"，名片上能点"看优惠""进群"。
- 在哪：后台 → 通讯录 → 对外展示，改个顾客看得见的名字、挂上团购/点评链接。
- 坑：名字别用真名或法人全名，用"店名+店长"。店员换人，换个人接着用，顾客无感。

**③ 设好"自动欢迎语"**
- 干啥：顾客一加上，自动弹一句"张姐你好～进群抢红包"，不用手动一个个发。
- 在哪：加客户 → 欢迎语，插入顾客昵称 + 一个钩子（进群送啥）。
- 坑：只弹一次、自动的。话别长，一句问候 + 一个进群方式，用当地话更亲切。它只是引人进群的中间一步。

**④ 弄个"进群码"**
- 干啥：一个**永远不变**的群码，群满 200 人自动开下一个群，你不用管。
- 在哪：客户联系 → 客户群 → 建群码。群名前加"进群看公告"，福利放群公告里，少刷屏。
- 坑：企业微信一个群最多 200 人。想让顾客点链接直接进（不扫码），得另买第三方系统——前期用码就够。

## 老板最关心：钱和坑
- **花多少钱**：企业微信 2000 个顾客以内免费，超了每人一毛/年（2 万人≈2 千、5 万人≈5 千）。几千顾客一年几百块，复购随便赚回来。
- **要不要认证**：想让顾客看到你店名（而非店员真名），花 300 块认证。用营业执照名字或自有商标，热门好听的花名多半早被人注册了，硬用审核不过。名字要想好，长久事。
- **要不要买系统**：先别买。用免费的把四步跑顺，跑通了嫌手动麻烦再问美团/第三方（一年一两千到两千多），更省事、能看数据，但前期不急。最要紧的是"让更多人加进来"，不是把后台配得多高级。

## 已经有老顾客群？这样搬
别一上来宣布"老群解散"（伤感情）。先在新群放个专属福利（如"进新群，下次来送一瓶饮料"），把带福利的图发老群、招呼大家扫码过来；看到老顾客愿意来了，再慢慢关老群，给缓冲。

## 起步自测清单（让老板逐条打勾）
全打勾 = 私域第一步扎实了，已经比大多数同行往前一步：
- [ ] 有一个顾客能扫的"加好友"码
- [ ] 店员名片顾客看着是"XX 店长"、能点进优惠和群
- [ ] 顾客加上会自动收到欢迎语 + 进群方式
- [ ] 有一个满了自动开新群的进群码
- [ ] 认证过了、顾客能看到你的店名
- [ ] 收银台有人提醒顾客"加微信进群有福利"

## 收尾（马甲式实在话）
> 别一上来就追求海报多好看、系统多高级。传统餐饮最缺的从来不是这些，是"落地"俩字。上过那么多课、听过那么多概念，卡住的往往就是"连后台第一步都没配"。先把人一个一个加进来，这步做到，地基就夯实了——上面想盖几层楼都有地方站。

## 产出形态二：交互导览网页（单文件 HTML）

一个 `<!doctype html>` 单文件、零外部依赖。要素：
- **CSS 变量做昼夜双主题**（暖橙 `#e8622c` + 奶白 `#faf7f2` + 墨黑 `#2b2723`；深色底 `#1c1917`）。`@media(prefers-color-scheme:dark)` + `[data-theme]` 手动切换都支持。
- **顶部**：标题"私域，到底怎么搭？"+ 一句话把"私域"翻成人话。
- **五步地图**：内嵌上面那段 SVG。
- **起步四件套**：做成可点击展开的手风琴（accordion），每条含 干啥/在哪弄/坑/一个自测勾选框。
- **钱和坑**：三张卡 + 一个花钱小表。
- **总自测**：6 项勾选清单 + 进度条 + 全勾满显示鼓励横幅；用 `localStorage` 存进度，下次打开还在。步内勾选和总清单**语义一一对应、双向联动**。
- **移动端友好**：老板多半手机看，SVG 容器 `overflow-x:auto`，卡片纵向堆叠。
- 页尾一行低调署名：`私域专家团 · 马甲实战版`。

参考成品结构：`~/Downloads/siyu-laoban-xiangdao/index.html`（马甲已验证的样板，可直接照它的骨架生成，替换成当前老板的品类/店名即可）。

## 什么时候升级到"深度定制版"
装了完整仓库（`src/siyu_team` 在、`siyu-onboard` 可用）且用户**是专业运营、要针对自家数据做全盘诊断**时，才走 `siyu-onboard` 的四官评审深度版。普通店老板问"怎么搭"，这份零依赖向导就够——**够用、能落地、看得懂，比一份跑不动的复杂编排强得多。**
