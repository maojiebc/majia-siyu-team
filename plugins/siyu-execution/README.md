# 私域执行层 · siyu-execution

一线私域运营**直接干活**的单技能，跟诊断层（四官专家团）平级。用户可以直接召唤某一个 skill，不必走团长编排。

> 为什么单独一层：一线运营每天要写朋友圈、发群、给话术，他要的是「一个直接出活的单专家」，不是被拉去做战略评审。通用「生成」市面已到 80 分，这一层补的是最后 20 分——**行业灵魂 + 真实 SOP + 边写边合规 + 落到门店真数据**。

## 已建 / 规划

| skill | 干什么 | 挂哪条方法论 | 状态 |
|---|---|---|---|
| `siyu-pyq` | 朋友圈文案（按配比+爆款结构出整周内容池，合规前置扫描） | 内容即产品 | ✅ 骨架已就位 |
| `siyu-qunfa` | 社群群发 / 栏目脚本（多人设语气、救打开率、偷着打折模板） | 运营即广告 | ✅ 骨架已就位 |
| `siyu-huashu` | 破冰欢迎 + 答疑话术库（分场景分人群，含造 IP 命名模板） | 私域即公关 | ✅ 骨架已就位 |

## 三个共用一套底层（马甲的差异化）
- **真实语料库** —— 各 skill 的 `assets/` 注入真实卖点/门店 SOP/客户见证（护城河）
- **合规规则包** —— `scripts/*_lint.py` 与质量门统一引用 `src/siyu_team/eval/compliance_lexicon.py`，**边写边合规**
- **三句话质检** —— 每个 skill 内置输出自检，用户无感却自带专家团 DNA

## 与诊断层（专家团）的关系
执行层 = 入口（高频、干活）；专家团 = 升舱（低频、拍板）。只有"帮我看整盘私域怎么搭 / 这个活动战略对不对"才路由到团长走四官评审。海报需求不在这里自造，指向 `guizang-social-card-skill` / `baoyu-cover-image`。
