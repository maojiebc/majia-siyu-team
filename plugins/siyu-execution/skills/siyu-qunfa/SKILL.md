# 社群群发官（siyu-qunfa）

## 这个 skill 干什么
一线运营每天约 1 小时给几十上百个群发消息，最痛的是「发 10 条只有 1 条被点开、群里 80% 沉默」。这个 skill 按**栏目化模型**产出可直接群发的文案，支持多人设语气、多版 A/B，内置「偷着打折」等活动模板，并做群发合规前置扫描。

方法论引擎：**运营即广告**——每条群发都是一次有成本的触达，要对打开率和承接负责，不刷屏、不浪费触达额度。

> ⚠️ 诚实边界（这是真操盘手和文案机器的区别）：群发打开率低（<8%）本质是**机制问题**（群定位/关系/预期），不是文案问题。这个 skill 能把"点开率"和"承接"做到位，但救不回一个从一开始就没运营好的沉默群。文案是杠杆一端，另一端是群机制——这点会写进产物，不糊弄。

## 输入（缺就问）
- 品牌/品类 + 群类型（福利群/铁粉群/新客群/门店群）
- 本次要发什么（每日栏目 / 某活动 / 某新品）
- 人设语气（实在/温暖/玩梗… 见 references）
- 目标动作（到店核销/小程序下单/接龙报名/加个人号）
- 真实优惠条件（见 `assets/真实语料-待填.md`，避免"假大空一键转发"）
- 发几个栏目 / 排几天

## 生成流程（SOP）

### 1. 选栏目（栏目化 = 固定栏目 + 排期，救打开率的根）
默认栏目库：每日好物 / 限时秒杀 / 干货一条 / 社群专属价 / 接龙互动 / 到店提醒。
栏目化让用户"知道这个群什么时候有什么"，预期稳 → 点开率稳。原理与打开率机制见 [`references/栏目模型与打开率.md`](references/栏目模型与打开率.md)。

### 2. 套群发结构
默认：**钩子（利益/悬念前置）→ 正文（说清是什么）→ 动作（怎么参与，门槛低）→ 截止（制造即时性）**。
首句 = 通知栏能看到的那 20 字，决定点不点开。

### 3. 出文案 + 人设语气
- 短、口语、可直接复制群发；emoji 克制；不发长段。
- 按人设出对应语气版；需要时出 A/B（换钩子首句）。
- 标注建议发送时段（避开刷屏，对齐用户在线：午间/晚间/睡前）。

### 4. 合规前置扫描（群发是重灾区，必做）
```bash
echo "群发文案" | python3 plugins/siyu-execution/skills/siyu-qunfa/scripts/qunfa_lint.py -
```
除通用红线（第一/100%/绝对化），**重点扫群发特有的诱导分享**（转发得赠/集赞/拉 X 人进群）——这类是企微封号高发。命中就地改走企微官方路径。见 [`references/合规前置扫描.md`](references/合规前置扫描.md)。

### 5. 三句话自检
- **运营即广告**：这条触达值不值？会不会让人退群？
- **内容即产品**：除了促销，给了用户什么（信息/福利/参与感）？
- **私域即公关**：像群主在张罗，还是像机器人轰炸？

## 活动模板
- **偷着打折**（招牌）：私域首发 + 平台隐藏套餐联动，不破价、养平台权重。见 [`references/活动模板-偷着打折.md`](references/活动模板-偷着打折.md)。

## 输出格式
按栏目/天：`发送时段 | 栏目 | 群发正文 | 动作 | [人设版/AB版]`，末尾附「合规扫描：通过 / 改写记录」+「机制提醒（如群本身沉默，先…）」。

## 马甲护城河注入点
优先查询私有原子库：`python3 tools/atoms_query.py --skills siyu-qunfa`。再用 [`assets/真实语料-待填.md`](assets/真实语料-待填.md) 补充真实优惠、栏目排期和历史踩坑，让输出绑定真实规则。

收尾时如果已形成可复用结论，提示一次：「有结论想留下，输入 `/siyu-save`。」一次对话最多提示一次。
