# 群发栏目模型 · 打开率机制 · 时段频次

## 一、栏目库（栏目化 = 固定栏目 + 排期）
| 栏目 | 作用 | 频率参考 |
|---|---|---|
| 每日好物 | 主推单品，稳定输出 | 每日 1 |
| 限时秒杀 | 制造即时性、拉下单 | 每周 1–2 |
| 干货一条 | 非促销，攒信任、降广告感 | 每周 2–3 |
| 社群专属价 | 让"在群里"有实际好处 | 每周 1 |
| 接龙互动 | 拉参与、探需求、破沉默 | 每周 1–2 |
| 到店提醒 | 饭点前/活动前的临门一脚 | 按需 |

栏目化的意义：用户"知道这个群什么时候有什么"→ 预期稳定 → 点开有理由。乱发 = 没预期 = 沉默。

## 二、群发结构（每条都套）
**钩子 → 正文 → 动作 → 截止**
- 钩子：利益或悬念前置，塞进前 20 字（通知栏只显示这段）。
- 动作：门槛越低越好（"报暗号""扣 1""点小程序"）。
- 截止：给即时性（"只限今天""前 30 位"），但别造假。

## 三、打开率的机制真相（诚实边界，写进产物）
群发打开率<8% 时，先分清是**文案问题**还是**机制问题**：
- 文案能救的：钩子太弱、没利益、太长、太硬广 → 本 skill 能提升点开与承接。
- 文案救不了的（机制问题，要先改机制）：
  - 群定位模糊（用户不知道这群是干嘛的）
  - 关系没建立（进群就被推销，没先给价值）
  - 预期被破坏（说好福利群天天发广告）
  - 群已死（长期无互动，先做小范围激活/重建，别继续群发）

> 真操盘手会先说这句：**"这个群该救文案还是该救机制"**——而不是一味出更花的文案。这是 siyu-qunfa 相对纯文案工具的诚实之处。

## 四、时段与频次
- 对齐在线：午间（12:00–13:00）、晚间（19:00–21:00）、睡前（22:00 前）。饭点前是餐饮的黄金触达。
- 频次：福利群每日 1–2 条足够；铁粉群可高些；**同一群一天别超 3 条**，否则免打扰/退群。
- A/B：同一条只换钩子首句，发不同群或隔天，看点开与动作转化。

## 五、人设语气（可切换参数）
- 实在（餐饮/门店）：像老板/店长在张罗，接地气不端着。
- 温暖（母婴/健康）：关怀式，多用"咱""给宝宝"。
- 玩梗（潮牌/年轻）：网感、短、带梗，但别硬玩。
