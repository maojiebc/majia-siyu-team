# 餐饮私域方法论（行业参考层）

> 综合自公开的餐饮私域实战内容与行业通行打法。
> 定位：可公开的「行业参考 / 方法论层」，不是真实护城河 SOP（那在 `knowledge/03-majia-sop/`，私有）。
> 用法：给各 skill 提供餐饮行业的默认框架；真实卖点/阈值/数据由使用者在 assets 或私有 SOP 层注入。

## 一、私域五步骨架
造 IP → 做链接 → 搞促活 → 做分层 → 拉复购。
- 私域 ≠ 加个人、拉个群；五步都运营透才有持续增收。
- 映射行业册阶段：冷启动 = 造 IP + 做链接；扩张 = 促活 + 分层；成熟 = 复购 + 规模化玩法。

## 二、造 IP 公式（第一步）
账号 = 头像 + 名称 + 朋友圈背景，三者都关联品牌。
- **头像**：真人 + 微笑 + 工服/门店场景；**禁用 logo**（除非品牌势能极强）。
- **名称模板**：单店 = `门店 + 人名（+ 福利发放版）`；连锁 = `品牌 + 门店 + 职务 + 人名`。反面：本名 / 法人全名（离店后没人找得到）。
- **朋友圈背景图** = 招牌特色菜。

## 三、3322 朋友圈内容法则（做链接）
餐饮朋友圈内容配比：
- **30% 美食展示**（真实出片，非菜单抠图）
- **30% 吃法案例**（搭配/蘸料，显专业）
- **20% 工作日常**（巡店/做饭/开会，养 IP 人设）
- **20% 优惠福利**（这才是广告）

反面：一发就是打折 → 被当“广告垃圾站”秒删。
素材类型库：护餐抽奖 / 新品试吃 / 做菜过程 / 新店筹备 / 公益夹带 / 产品场景化。

> 已设为 `siyu-pyq`（朋友圈文案 skill）的餐饮默认配比。

## 四、招牌玩法卡：偷着打折
原则：**所有活动私域首发，平台联动**。动作链：
1. 平台后台（团购平台）建一个折扣力度比现有稍大的套餐。
2. 建完**不上架** → 走客户经理审核通过后**隐藏**。
3. 隐藏后：普通消费者从门店主页看不到这个专属优惠。
4. 隐藏套餐二维码做成海报 + 文案，发私域（群/朋友圈/1v1）。
5. 私域好友享“专属特权”下单，走平台隐藏套餐，每笔计入平台热度值。

好处：① 公私域交易闭环；② 不打乱平台价格体系（不破价）；③ 养平台权重。

> 落地：成熟阶段的玩法卡 + `siyu-qunfa` 的活动模板。合规注意：优惠表述走真实条件，海报文案过合规扫描。

## 五、团购套餐诊断维度
上传套餐/榜单/平台扣点/毛利，可诊断：套餐数是否达平台评分标准、同类套餐内部分流、缺哪类搭配、定价建议（对齐人均）、引流款/利润款/代金券结构、五连图/头图/进店转化率、价格带分层（单人/双人/多人聚会）。

## 六、会员成熟度误区
“做了会员 ≠ 做好会员；有私域 ≠ 玩好私域”。多数品牌只看“有多少会员、复购多少”，从不挖**消费心智/频次/喜好**；生意一波动就发券/发红包/找达人，而不是激活存量。
